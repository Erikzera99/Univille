HTML Semântico

HTML Semântico
e sua importância

Tags semânticas comunicam significado — não apenas aparência. Beneficiam leitores de tela, SEO e manutenibilidade do código.

Principais tags semânticas

<header>

Cabeçalho de página ou seção. Contém título, logo e navegação.

<nav>

Bloco de links de navegação. Anunciado por leitores de tela.

<main>

Conteúdo principal único da página. Um por documento.

<section>

Agrupa conteúdo temático. Deve ter um título (h2-h6).

<article>

Conteúdo autossuficiente que faz sentido sozinho.

<figure>

Conteúdo ilustrativo (imagem, código) com legenda opcional.

A11Y

Acessibilidade
no código HTML

Acessibilidade (A11Y) garante que pessoas com deficiências possam usar o site. É obrigatória por lei em muitos países e beneficia todos os usuários.

01

ARIA Labels

aria-label, aria-labelledby e aria-describedby fornecem contexto para leitores de tela quando o texto visível não é suficiente.

02

Foco visível

Todo elemento interativo deve ter um anel de foco visível. Usamos :focus-visible para exibir apenas quando há navegação por teclado.

03

Skip Link

O link "Pular para o conteúdo" (visível ao pressionar Tab) permite usuários de teclado pularem a navegação repetitiva.

04

Alt text

Imagens decorativas usam aria-hidden="true". Imagens informativas têm alt descritivo ou role="img" com label.

05

Hierarquia de títulos

h1 → h2 → h3 em ordem. Não pule níveis. Leitores de tela usam a hierarquia para navegar pela página.

06

Contraste de cores

WCAG 2.1 exige contraste mínimo de 4.5:1 para texto normal e 3:1 para texto grande. Nosso esquema atende esse critério.

CSS Grid —
layout bidimensional

Flexbox vs Grid — quando usar cada um?

⟷ Flexbox

  • Uma dimensão (linha OU coluna)
  • Navegação, toolbars, menus
  • Centralizar um elemento
  • Alinhar ícone + texto
  • Distribuir itens numa linha

⊞ CSS Grid

  • Duas dimensões (linha E coluna)
  • Layout de página completo
  • Galerias de imagens
  • Grids irregulares (spans)
  • Dashboard com múltiplos painéis
Mobile First

Responsividade
e Mobile First

Mobile First significa escrever o CSS base para telas pequenas e adicionar regras para telas maiores com min-width. Resulta em código mais limpo e prioritiza a maioria dos usuários.

📱
Mobile (base) Layout em coluna única, menu hambúrguer
0 — 575px
📲
Small — sm Nav visível, grids de 2 colunas, menu oculto
≥ 576px (36rem)
💻
Medium — md Layout de 2 colunas, rodapé 3 colunas
≥ 768px (48rem)
🖥️
Large — lg Grid de revista, layout de 3 colunas complexo
≥ 1024px (64rem)
🖥️
Extra Large — xl Rodapé 4 colunas, container máximo
≥ 1280px (80rem)
styles.css — abordagem Mobile First 
/* ✅ MOBILE FIRST — Base: tela pequena (regra padrão) */
.nav-lista {
  display: none;  /* escondida no mobile */
}

.grid-layout {
  grid-template-columns: 1fr;  /* 1 coluna no mobile */
}

/* 📲 Tablet (≥ 576px) — adicionamos regras */
@media (min-width: 36rem) {
  .nav-lista {
    display: flex;  /* aparece a partir do sm */
  }
  .grid-layout {
    grid-template-columns: 1fr 1fr;  /* 2 colunas */
  }
}

/* 🖥️ Desktop (≥ 1024px) — layout completo */
@media (min-width: 64rem) {
  .grid-layout {
    grid-template-columns: 2fr 1fr 1fr;  /* layout de revista */
  }
}
Animações CSS

Animações
com CSS puro

@keyframes + animation criam movimento sem JavaScript. Combinado com transition em hover, resultam em interfaces vivas e expressivas.

Pulsar scale + opacity

animation: pulsar 1.5s ease-in-out infinite
Útil para indicar estado ativo ou atrair atenção.

Girar rotate 360°

animation: girar 2s linear infinite
Ideal para loaders, ícones de carregamento.

Deslizar translateX

animation: slide 2s ease-in-out infinite alternate
Movimento de banner ou slider.

Flutuar translateY + rotate

animation: flutuar 3s ease-in-out infinite
Efeito orgânico para elementos decorativos.

Fade opacity + scale

animation: fade 2s ease-in-out infinite alternate
Suave aparecimento/desaparecimento.

Escala scale + rotate

animation: escala 1.8s ease-in-out infinite alternate
Combinação de escala e rotação.

Medidas CSS

Medidas fixas
vs relativas

Medidas relativas (rem, em, %, vw) se adaptam ao contexto. Medidas fixas (px) são rígidas. Use relativas para layouts responsivos e acessíveis.

Redimensione a janela e veja o texto se adaptar:

Título Principal

font-size: clamp(1.5rem, 5vw, 3rem)

Subtítulo de Seção

font-size: clamp(1.2rem, 3.5vw, 2rem)

Título de Card

font-size: clamp(1rem, 2.5vw, 1.5rem)

Texto de parágrafo — fluido entre mobile e desktop sem media queries!

font-size: clamp(0.85rem, 1.5vw, 1.1rem)

❌ Medidas Fixas (px)

Não respeitam preferências do usuário. Podem quebrar o layout em telas incomuns.
